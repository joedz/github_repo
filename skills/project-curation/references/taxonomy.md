# Taxonomy Reference

Use stable English labels for directories and filenames. Keep the actual saved path compact and predictable.

## Category Suggestions

Choose the closest existing label first.

| Chinese | English slug | Use for |
|---|---|---|
| AI 工具 | `ai-tools` | AI products, model tooling, agent systems, prompt or workflow tools |
| 前端 | `frontend` | Web UI, design-heavy apps, component systems, visualization |
| CLI 工具 | `cli` | Terminal tools, developer automation, command-line workflows |
| 基础设施 | `infra` | Deployment, DevOps, platform engineering, observability |
| 后端 | `backend` | APIs, services, server frameworks, data processing backends |
| 数据工具 | `data-tools` | ETL, analytics tooling, data pipelines, data apps |
| 移动端 | `mobile` | iOS, Android, React Native, Flutter apps |
| 游戏 | `games` | Game engines, game tooling, interactive play experiences |
| 机器人/自动化 | `automation` | Bots, workflow automation, browser or agent automation |
| 教育 | `education` | Learning products, teaching tools, study systems |

## Feature Suggestions

Choose the main reason the project deserves saving.

| Chinese | English slug | Use for |
|---|---|---|
| 交互体验 | `ux` | Flow design, polished interactions, clarity, delight |
| 代码结构 | `code-architecture` | Module boundaries, clean layering, maintainable structure |
| 工程化 | `developer-experience` | Tooling, setup quality, docs, scaffolding, local workflow |
| 性能优化 | `performance` | Rendering speed, memory efficiency, runtime optimization |
| 视觉设计 | `visual-design` | Typography, layout, styling, visual identity |
| API 设计 | `api-design` | Clean interfaces, composability, SDK ergonomics |
| 可扩展性 | `extensibility` | Plugin systems, hooks, customization surfaces |
| 测试质量 | `testing` | Strong test coverage, test ergonomics, reliability patterns |
| 可观测性 | `observability` | Logging, tracing, dashboards, diagnostics |
| 文档表达 | `documentation` | Clear docs, examples, onboarding, teaching quality |

## Naming Rules

- Prefer the table values above when they fit
- Use lowercase kebab-case only
- Avoid synonyms that fragment the collection
- If none fit, add one new label and keep it broad enough to reuse later

## Tie-Breaking

If multiple categories fit, choose the one that best matches the project's main user-facing purpose.

If multiple features fit, choose the reason you would most likely remember the project six months later.
